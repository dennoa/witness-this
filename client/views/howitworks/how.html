How it Works in General
  • Ethereum is a P2P network and we need to run an Ethereum node to join in.
  • We run a full node that stores the entire blockchain locally and constantly synchronises with other peers. Light client support is not there yet.
  • We are identified on the network by our Ethereum addresses. We give our node permission to send messages from our address and spend our Ether.
  • The Ethereum node validates your address and password against a keystore before allowing the transaction to continue.
  • We interact with our node over HTTP via its API. No SSL support so interactions are local. We run a web server alongside the Ethereum node to accept remote requests.
  • We minimise data storage on the blockchain by only storing a hash reference that can be used to verify the data integrity. The actual data is stored elsewhere.
Specifically for this POC
  1. Upload a set of documents to a web server (NodeJS).
  2. Create hash representations of each documents content (sha256).
  3. Combine document hashes and timestamps for each document in the set and hash the combined data.
  4. Save the data to our own database (not the actual files, just the identifying data).
  5. Record the combined hash on the blockchain (Ethereum Smart Contract). This updates blockchain data so is slow and costs Ether. (The update needs to be accepted by the other nodes).
  6. Later on when we want to verify a document, we upload the current version.
  7. We retrieve the relevant set of document information from our own database and recalculate the combined hash.
  8. We verify the combined hash against the blockchain (Ethereum Smart Contract). This reads data from the local blockchain so is fast and doesn't cost any Ether.
  9. If the integrity of the data set remains intact, we verify the current file hash against the one previously stored in our own database.
